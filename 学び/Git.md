## Git


### トラブル事例
1. pullしたときに、"チェックアウトの前に、リポジトリの作業ツリーを消去してください"とエラーが表示される。
    - 何を示しているのか
        - pull(内部的には、fetch → merge/rebase)しようとしたときに、ローカルリポジトリに未コミットの変更があるため、Gitがその変更を上書きしてしまう可能性があると判断し、処理を中断したことを示している。
        - つまり、まだ変更をステージングにコミットしてないのに、リモートの内容を上書きしようとしていることを怒られている
    - 画面に表示されている選択肢
        - Git ログを開く
            - コミット履歴ビューが開く
            - 技術的な詳細を確認したい場合に有用
        - コマンド出力を表示する
            - Gitコマンドの実行ログが表示される
    - 解決のために
        - ローカルの変更を確定させる（コミット）
            - stagingにコミットし、pull
            - 基本はこっち
        - ローカルの変更を一時的に退避させる（スタッシュ）
            - まだコミットしたくない一時的な変更がある場合に便利
            - stash save "comment"→ pull → stash pop
