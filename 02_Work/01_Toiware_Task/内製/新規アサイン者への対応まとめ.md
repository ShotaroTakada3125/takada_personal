# 新規アサイン者対応まとめ

## 1. 事前準備
- 必要なアカウント・権限の洗い出し
- プロジェクト概要説明・オリエンテーション
- 環境構築資料の整備・修正

## 2. アカウント発行・権限付与
- 下記はシステム部の加藤さん、白濱さん宛に依頼
    - AWS（STG/PROD等）
    - 開発VPN
    - Redmine
    - Box
    - Asana
    - Google Drive(VCTパートナー共有)
        - 必要に応じてBoxへ移行
    - GitHub（simon-arg/custteam, simon-arg/toiware）
    - 「テスト口座用メール受信箱」Slack チャンネル

- 下記は既存開発メンバーやテストメンバーで対応可能
    - BOWへのアカウント追加
    - リセットバッチ追加
    - Trustdockへのログイン情報

## 3. 初回オリエンテーション

- チーム体制・連絡手段（Slack等）の案内
    - wiki修正で対応
