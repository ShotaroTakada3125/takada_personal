# コンヴァノ株主優待申込キャンセル機能 仕様書

## 1. 概要
株主優待の申込状況を確認し、紐付けられた口座情報を解除（キャンセル/リセット）するための管理画面機能

今回はコンヴァノ株主優待申込キャンセルの画面と処理、ロジックについて調査した

## 2. 画面仕様

- 画面名: コンヴァノ株主優待申込キャンセル
- URLパス: `/convano-stock-holder-bonus-cancel`
- ファイル: `module/inhouse-client/src/views/ConvanoStockHolderBonusCancel.vue`

### 2.1 検索条件
株主番号または口座番号のいずれかで検索が可能

- 検索の項目
    - 株主番号：`stockHolderNumber`
    - 口座番号：`accountId`

- 株主番号、口座番号ともに入力バリデーション:`入力がある場合8桁であること` があり、エラーメッセージを返す
    - なお、ここでは使用してないがクーポン番号は14桁となっている


### 2.2 検索結果表示項目
後述のAPIから申し込み情報を取得した場合、申込情報を表示する

| 項目名 | 物理名 |
| :--- | :--- |
| 株主番号 | `stockHolderNumber` |
| 申込パスワード | `entryPassword` |
| 紐付けられた口座番号 | `accountId` |
| 申込開始日時 | `entryStartDateTime` |
| 申込終了日時 | `entryEndDateTime` |
| 対象年月 | `term` |

### 2.3 操作ボタン

画面に表示されるボタンと、押下時のアクション、それぞれのボタンが表示される条件

- 検索ボタン
    - 実行関数：`getConvanoStockHolderBonus()`
    - 常に表示される
- キャンセル実行ボタン
    - 実行関数：`resetConvanoStockHolderBonus()`
    - 検索結果に、口座番号が存在する場合にのみ表示される

## 4. 処理フロー詳細

### 4.1 申込情報取得処理 (検索)
- ユーザーが検索ボタンを押下した時の処理フロー

1.  APIリクエスト:
    - triger：「検索」ボタンを押した時
    - Method: `POST`
    - URL: `/inhouse/convano-stock/get`
    - Request Body: `ConvanoStockHolderBonusReq`
        - 株主番号, 口座番号

2.  サーバー側処理 (`Service.get`):
    - 株主番号と口座番号が両方空の場合は `null` を返却
    - ロジック側データ取得 (`Logic.get`):
        - リポジトリにおけるSQLの実行
            - `@Query` により定義されたJPQLが実行される
            - `Campaign`テーブルと結合し、削除されていないデータを検索
        - 実行メソッド: `findByStockHolderNumberOrAccountId`
    - キャンペーン期間チェック:
        - 取得したデータの `campaignId` を元にキャンペーン情報を取得
            - テーブル名：`campaign`
        -   現在日時がキャンペーンの掲載期間 (`PublishFrom` ～ `PublishTo`) 外であれば、`null`を返却

3.  レスポンス:
    - 対象データが存在し、期間内であれば `ConvanoStockHolderBonusRes` を返却
    - データなしの場合はフロントでエラー文言表示

### 4.2 申込キャンセル処理
ユーザーがキャンセル実行ボタンを押下した時の処理フロー
キャンセル実行は、検索結果に口座番号が返ってきた場合のみ実行可能


1.  APIリクエスト:
    - triger：「キャンセル実行」ボタンを押した時
    - Method: `POST`
    - URL: `/inhouse/convano-stock/cancel`
    - Request Body: `ConvanoStockHolderBonusReq` (検索結果に含まれていた 株主番号, キャンペーンID, 申込開始日時, 申込終了日時, 対象年月)

2.  サーバー側処理 (`Service.resetAndGet`):
    - データ再取得: 主キー（株主番号 + キャンペーンID）で対象レコードを特定存在しない場合はエラー
    - ログ出力: 更新前後の状態をログ出力
    - ロジック側リセット実行 (`Logic.resetEntity`):
        - 対象エンティティの `accountId` (口座番号) を `null` に設定
    - 保存処理（リポジトリ）: `saveAndFlush` でDBを更新
      - save: エンティティの変更内容を保存キューに入れる
      - Flush: DBに即座に書き込むSQLを発行し、コミット

3.  レスポンス:
    - 更新後のエンティティを `ConvanoStockHolderBonusRes` に代入
    - フロントは画面情報を更新後のデータで再描画し、完了メッセージを表示
