# 新規管理画面機能 開発ガイド

## 1. フロントエンド (Vue.js)

### 1.1 作成・修正ファイル
| 種類 | パス (module/inhouse-client/src/) | 役割 |
| :--- | :--- | :--- |
| **画面コンポーネント** | `views/【機能名】.vue` | 画面の見た目と挙動を実装 (Vueファイル) |
| **ルーティング定義** | `router.js` | 画面へのURLパスを追加 (`routes`配列に追加) |
| **リクエストモデル** | `models/request/【機能名】Req.js` | APIへ送るパラメータの定義クラス (必要であれば) |
| **API定義** | `models/【機能名】.js` | APIのエンドポイント呼び出し処理 (共通化する場合) |

**手順概要:**
1. `views/` に `.vue` ファイルを新規作成する。
2. `router.js` に新しいパスとコンポーネントを登録する。
3. 必要に応じてモデルクラスを作成する。

## 2. バックエンド (Spring Boot / Java)

基本的なレイヤーアーキテクチャに従い、以下の4層を作成します。

### 2.1 Controller (Web層)
APIのエンドポイント（入口）を定義します。

- **ファイルパス**: `module/inhouse-app/src/main/java/jp/co/toiware/cust/application/controller/【機能名】Controller.java`
- **アノテーション**: `@RestController`, `@RequestMapping`
- **役割**: リクエストを受け取り、Serviceを呼び出し、レスポンスを返す。

### 2.2 Service (Business層)
業務フロー、トランザクション管理、バリデーションを行います。

- **ファイルパス**: `module/core/src/main/java/jp/co/toiware/cust/application/service/【機能名】Service.java`
- **アノテーション**: `@Service`, `@Transactional`
- **役割**: Logicを組み合わせて業務を実現する。

### 2.3 Logic (Logic層)
具体的なビジネスロジック、データ加工、Repositoryの呼び出しを行います。

- **ファイルパス**: `module/core/src/main/java/jp/co/toiware/cust/domain/logic/【機能名】Logic.java`
- **アノテーション**: `@Component`
- **役割**: Repositoryを使ってデータを取得・保存する具体的な処理を書く。

### 2.4 Repository (Data Access層)
データベース操作を行います。

- **ファイルパス**: `module/core/src/main/java/jp/co/toiware/cust/domain/primary/repository/【機能名】Repository.java`
- **アノテーション**: `@Repository`
- **継承**: `JpaRepository` 等
- **役割**: SQL（JPQL）を記述し、DBアクセスを行う。

### 2.5 Entity / DTO (Data層)
データの入れ物です。

- **Entity**: `module/core/src/main/java/jp/co/toiware/cust/domain/primary/entity/【機能名】.java` (DBテーブル定義)
- **Request DTO**: `module/core/src/main/java/jp/co/toiware/cust/application/dto/【機能名】Req.java` (入力データ)
- **Response DTO**: `module/core/src/main/java/jp/co/toiware/cust/application/dto/【機能名】Res.java` (出力データ)

---

## 3. 開発フローの推奨順序

1. **DB設計 & Entity作成**: まず扱うデータを定義します。
2. **Repository作成**: データの読み書き部分を作ります。
3. **Logic & Service作成**: 業務ロジックを実装します。
4. **Controller作成**: APIとして公開します。
5. **Frontend作成**: 作成したAPIを呼ぶ画面を作ります。
6. **Router登録**: 画面が見えるようにします。
