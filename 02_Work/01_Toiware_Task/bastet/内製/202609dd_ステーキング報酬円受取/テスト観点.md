# ステーキング報酬円受取
## テスト観点

### DB
- staking_rewards_conversion_entry テーブル
  - テーブルが作成されていること
  - 以下カラムが全て存在すること
    - 論理名：物理名
    - 管理番号：id
    - 口座番号：account_id
    - 通貨名：currency

  - jpy_conversion_enabled カラムが追加されていること

  - 指定の銘柄が true であること

### API
- 円受取対象銘柄一覧取得API
  - 異常系 (開発者確認)
    - 不正な口座番号でエラーが返ることを確認
      1. APIを実行
        ```bash
        curl -i -X GET "http://localhost:21002/staking/conversion/99999999"
        ```
      **期待結果:**
      - HTTPステータス: `404 Not Found`
      - エラーメッセージ: {"error":1004,"message":"指定された口座は登録されていません。"}
    - 口座開設完了/本登録以外のステータスで空リストが返ることを確認
      1. APIを実行（仮口座の口座番号を使用）
        ```bash
        curl -i -X GET "http://localhost:21002/staking/conversion/10000002"
        ```
      **期待結果:**
      - HTTPステータス: `200 OK`
      - レスポンス: `[]` (空配列)
- 円受取申請登録API
  - 異常系（開発者確認）
    - 円受取対象でない銘柄を指定した場合のエラー処理を確認
      1. APIを実行（BTCは円転対象外と仮定）
        ```bash
        curl -i -X POST "http://localhost:21002/staking/conversion/register" \
          -H "Content-Type: application/json" \
          -d '{
            "accountId": "10000008",
            "changeList": [
              {
                "currency": "BTC",
                "jpyConversionEnabled": true
              }
            ]
          }'
        ```
      **期待結果:**
      - HTTPステータス: `400 Bad Request`
      - エラーメッセージ: {"error":1000,"message":"パラメーターが正しくありません。"}

    - 口座番号が不正な状態でのエラー処理を確認
      1. 不正な口座番号でAPIを実行
        ```bash
        curl -i -X POST "http://localhost:21002/staking/conversion/register" \
          -H "Content-Type: application/json" \
          -d '{
            "accountId": "123",
            "changeList": [
              {
                "currency": "ETH",
                "jpyConversionEnabled": true
              }
            ]
          }'
        ```
      **期待結果:**
      - HTTPステータス: `400 Bad Request`
      - バリデーションエラーメッセージ {"error":1000,"message":"accountId must match \"\\d{8}\""}
    - 必須パラメータが欠けている場合のエラー処理を確認
      1. changeListを送信しない
        ```bash
        curl -i -X POST "http://localhost:21002/staking/conversion/register" \
          -H "Content-Type: application/json" \
          -d '{
            "accountId": "10000001"
          }'
        ```
      **期待結果:**
      - HTTPステータス: `400 Bad Request`
      - {"error":1000,"message":"changeList must not be null"}
    - 仮登録など口座ステータスが不正な場合のエラー処理を確認
      1. 仮口座の口座番号でAPIを実行
        ```bash
        curl -i -X POST "http://localhost:21002/staking/conversion/register" \
          -H "Content-Type: application/json" \
          -d '{
            "accountId": "10000002",
            "changeList": [
              {
                "currency": "ETH",
                "jpyConversionEnabled": true
              }
            ]
          }'
        ```
      **期待結果:**
      - HTTPステータス: `417 Expectation Failed`
      - エラーコード: `INVALID_ACCOUNT_STATE`
      - {"error":2008,"message":"口座状況が不正です。"}

### フロントエンド
- マイページに 「ステーキング報酬の受取方法」表示領域があること
  - figmaとデザインに相違がないこと
  - 口座開設完了、本登録以外のステータスでは表示されないこと
- 「ステーキング報酬の受取方法」動作
  - 正常系
    - 通貨ごとに 「通貨受取 / 円受取」が選択できること
    - 「更新する」ボタンを押したことがないアカウントでマイページを開くと、全て「暗号資産」がデフォルトで銘柄一覧が表示されていること
    - 「更新する」ボタンを押したことがあるアカウントでマイページを開くと、受け取り方法の設定が正しく反映された状態で表示されていること
  - 「更新する」ボタン押下
    - ボタン押下するたびに「ステーキング報酬の受取方法を更新しました。」のダイアログが表示されること
    - (開発者確認)DBのstaking_rewards_conversion_entry テーブル に設定が反映されていること
      - 初回ボタン押下で、円受取対象銘柄の設定全てが反映されていること
      - 2回目以降で、同じ月内に設定を変更した場合、既存レコードが上書きされることを確認
      - 2回目以降で、月をまたいで設定を変更した場合、新規レコードが作成されることを確認
      - 設定を変更しない場合、レコード更新/追加なし
    - 複数銘柄の一括更新ができること
  - 「ステーキング報酬の受取方法を更新しました。」ダイアログを閉じると変更が反映されていること
  - 設定を現在のものから異なるものを選択し、「更新する」ボタンを押さずにページから離れると、選択したものではなく、設定された状態になっていること

- ローディングスピナー表示

    - 「ステーキング報酬の受取方法」の設定が画面に表示されるまでに崩れる、異なる設定が表示されるなど
  - 「更新する」ボタン押下でローディングスピナーが表示されること
