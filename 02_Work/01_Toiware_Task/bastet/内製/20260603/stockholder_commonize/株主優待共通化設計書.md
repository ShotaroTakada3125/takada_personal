バックエンド (signup-api)
	いずれのAPIも共通仕様として、株主優待を特定するためのパスパラメータを設定する。
	keywordとtermをもとに株主優待マスタを特定する。
	またその株主優待マスタのIDとaccountIdをもとに、株主優待申込情報の特定する。
		基本URL							/stockholder_bonus_entry/{keyword}/{term}/{accountId}
		パスパラメータ							keyword						株主優待マスタの識別ワードに該当する
									term						株主優待マスタの対象期間に該当する
									accountId						申込者の口座番号

		keywordとtermに紐付く株主優待マスタが存在しない場合はエラーを返却する。
			エラーメッセージ「開催中の株主優待ではありません。」

	① 申込可否チェックAPI
		URL									/stockholder_bonus_entry/{keyword}/{term}/{accountId}/account_check
		HTTPメソッド									GET
		リクエストパラメータ									なし
		レスポンスデータ									株主優待名
											申込コードの項目名
											申込パスワードの項目名
											対象口座

		申込可能か否かを、以下の順にチェックする。
		・口座情報確認
			DBに存在している口座番号かを確認し、存在していない場合はエラーを返却する。
				エラーメッセージ「指定された口座は登録されていません。」
		・顧客種別確認
			株主優待マスタの対象口座の値が1(個人のみ)で、申込口座の顧客種別が法人の場合はエラーを返却する。
				エラーメッセージ「本キャンペーンは個人口座向けです。」
		・口座状況確認
			株主優待マスタの対象口座の値が2(制限なし)、かつ口座状況が5(開設完了)もしくは6(本登録)以外の場合、
			もしくは株主優待マスタの対象口座の値が1(個人のみ)、かつ口座状況が6(本登録)以外の場合はエラーを返却する。
				エラーメッセージ「お客様は口座開設未完了です、手続きを進めてください。」
		・株主優待申込済み確認
			株主優待マスタに紐付く株主優待申込情報に口座番号が登録済みである場合はエラーを返却する。
				エラーメッセージ「お客様はすでに{0}をお申込み済みです。\r\n{0}は1口座あたり1回までとなっております。」
				{0} : 株主優待マスタの株主優待名

		問題なければ、以下の項目をレスポンスに設定する。
			株主優待名										株主優待マスタの株主優待名
			申込コードの項目名										株主優待マスタの申込コードの項目名
			申込パスワードの項目名										株主優待マスタの申込パスワードの項目名
			対象口座										株主優待マスタの対象口座

	② 入力値検証API
		URL									/stockholder_bonus_entry/{keyword}/{term}/{accountId}/entry_check
		HTTPメソッド									POST
		リクエストパラメータ									申込コード
											申込パスワード
		レスポンスデータ									確認画面表示用の申込者氏名

		入力値が妥当であるかを、以下の順にチェックする。
		・該当株主優待の申込期間確認
			申込コードと申込パスワードが一致する株主優待場合はエラーを返却する。
				エラーメッセージ「お申込み受付期間外です。」

		・該当する株主優待申込情報データの存在確認
			申込コードと申込パスワードが一致する株主優待申込情報がない場合はエラーを返却する。
				エラーメッセージ「※{0}または{1}が不一致です。再度入力をお願いします。」

				{1} : 株主優待マスタの申込パスワード

		・該当する株主優待申込情報データの申込可能確認
			申込コードと申込パスワードが一致する株主優待申込情報の口座番号が登録済みであった場合はエラーを返却する。
				エラーメッセージ「入力された{0}、{1}はすでにお申込み済です。」
				{0} : 株主優待マスタの申込コード
				{1} : 株主優待マスタの申込パスワード

		問題なければ、以下の項目をレスポンスに設定する。
			確認画面表示用の申込者氏名												口座情報の氏名

	③ 株主優待申込API
		URL									/stockholder_bonus_entry/{keyword}/{term}/{accountId}
		HTTPメソッド									POST
		リクエストパラメータ									申込コード
											申込パスワード
		レスポンスデータ									なし

		①②のチェックを行い全て問題なければ、該当する株主優待申込情報の口座番号をaccountIdで更新する。
		登録後、申込者の口座情報のメールアドレスに対して申込み完了通知のメールを送信する。
		使用するメールテンプレートは、株主優待マスタの申込完了メールのテンプレートIDで定義しておく。

	④ 申込済みデータ取得API
		URL									/stockholder_bonus_entry/{keyword}/{term}/{accountId}
		HTTPメソッド									GET
		リクエストパラメータ									なし
		レスポンスデータ									登録状況メッセージ
											申込コード
											申込パスワード
											更新日時

		申込済みデータ確認の前に、以下のチェックを行う。
		・口座情報確認
			DBに存在している口座番号かを確認し、存在していない場合はエラーを返却する。
				エラーメッセージ「指定された口座は登録されていません。」
		・顧客種別確認
			株主優待マスタの対象口座の値が1(個人のみ)で、申込口座の顧客種別が法人の場合はエラーを返却する。
				エラーメッセージ「本キャンペーンは個人口座向けです。」

		全てのチェックが問題なければ、株主優待マスタに紐付く株主優待申込情報をaccountIdで検索し、
		申込済みデータの有無により以下の値をレスポンスに設定し返却する。
		・申込済みの場合
			登録状況メッセージ										「{0}をお申込み済みです」
													{0} : 株主優待マスタの株主優待名
			申込コードの項目名										株主優待マスタの申込コードの項目名
			申込パスワードの項目名										株主優待マスタの申込パスワードの項目名
			申込コード										株主優待申込情報の申込コード
			申込パスワード										株主優待申込情報の申込パスワード
			氏名										口座情報の氏名
			申込日時										株主優待申込情報の更新日時 (yyyy/MM/dd hh:mm形式)

		・未登録の場合
			登録状況メッセージ										「{0}にはまだお申込みいただいておりません」
													{0} : 株主優待マスタの株主優待名
			申込コードの項目名										株主優待マスタの申込コードの項目名
			申込パスワードの項目名										株主優待マスタの申込パスワードの項目名
			申込コード										ー
			申込パスワード										ー
			氏名										口座情報の氏名
			申込日時										ー

フロントエンド (signup-frontend)

	URLパスパラメータ設計
		各画面URLは `/{keyword}/{term}/{page}` のパターンとし、以下の流れで各キャンペーンを識別する
			keyword: 株主優待マスタの識別ワード（英数字・ハイフンのみ）
			term: 株主優待マスタの対象期間（数字のみ）
			page: 画面種別（input, entry, confirm, check, refer）
			例）
				/sbils/202606/input
				/sbils/202606/entry
				/sbils/202606/confirm
	各URLのパスパラメータ値の/{keyword}/{term}が、バックエンドのAPI呼び出し時に用いる値となり、
	対象の株主優待が決定する。

	① 株主優待申込ログイン (/{keyword}/{term}/input)
		株主優待申込用のログイン画面。

		入力項目：メールアドレス（口座番号）、パスワード
		処理フロー
			1. 口座番号（メールアドレス）とパスワードを入力し、ボタン押下
			2. simplexのログインAPIでログイン処理
			3. ログイン成功後、API①を実行
				API①がOK : 2要素認証画面へ遷移
				API①がNG : エラーアラート表示、エラーコードによって画面遷移
			4. 2要素認証成功後、画面②へ遷移

		各APIのエラー時の挙動については`機能詳細`に記載

	② 株主優待申込入力画面（/{keyword}/{term}/entry）
		株主優待申込みを行うための画面。
		各株主優待・キャンペーンで処理や文言が異なるため、共通化せず各株主優待で都度作成

		入力項目：申込コード、申込パスワード

		バリデーション
			サーバー検証：API②で実施
			フロント検証：文字種、桁数、注意事項への同意
				文字種や桁数、注意事項への同意チェックボックスのバリデーションチェックについては
				共通のバリデーションチェックを実施し、共通のバリデーション文言を表示する
					※バリデーション文言は`機能詳細`のエラーハンドリング項目を参照

		処理フロー
			1. 画面描画時：API①を実行
				API①がNG : エラーアラート表示、エラーコードによって画面遷移
			2. 各項目を入力し、「確認画面へ」ボタン押下
			3. クライアント検証：NGであれば入力欄にエラー表示
			4. クライアント検証OK：API②を実行
				API②がOK : 画面③へ遷移
				API②がNG : ボタン上にエラーメッセージ表示

	③ 株主優待申込確認（/{keyword}/{term}/confirm）
		表示項目: カスタム可能
			申込コード（項目名は API①のレスポンス `entryCodeName` より）
			申込パスワード（項目名は API①のレスポンス `entryPasswordName` より）
			メールアドレス（ログイン時のID）
			氏名（API②のレスポンス `fullName` より）
			その他キャンペーン固有項目（オプション）

		処理フロー
			1. 画面描画時：API①を実行
				API①がNG : エラーアラート表示、エラーコードによって画面遷移
			2. 「申込む」ボタン押下でAPI③実行
				API③がOK : 確認メッセージをアラート表示 → 通常ログイン画面へリダイレクト
					確認メッセージはデフォルト：「{0}のお申込みを受け付けました。」({0} : API①のレスポンス情報の株主優待名)
					カスタム可能
				API③がNG : ボタン上にエラーメッセージ表示

	④ 株主優待申込照会ログイン画面 （/{keyword}/{term}/check）
		内容は画面①と同様。
		二要素認証も成功した場合は、⑤の入力画面へ遷移する。
		画面①で表示していた注意事項や説明用コンポーネントは、画面④には表示しない。

	⑤ 株主優待申込照会確認画面（/{keyword}/{term}/refer）
		表示項目
			登録状況メッセージ
			申込コード（未登録時は「ー」）
			申込パスワード（未登録時は「ー」）
			メールアドレス
			氏名(法人名)   （個人のみの場合は `氏名`）
			申込日時（未登録時は「ー」）

		処理フロー
			1. 画面描画時：API④を実行
				API④がNG : 確認メッセージをアラート表示 → 通常ログイン画面へリダイレクト
				API④がOK : 表示項目にレスポンス内容を表示

	共通仕様
		画面②, ③, ⑤の画面遷移時における共通仕様
		・認証確認
			二要素認証を行なっているかクライアントサーバ側でセッションの値を確認し、
			URLで直接遷移するような画面遷移をした場合に、通常ログイン画面へ遷移する。

		・不正遷移確認
			画面遷移した際に、ログインした口座の口座状況を確認し、取得できなかった場合（URL直リンクでの遷移）は
			画面①の申込ログイン画面へ遷移する。

		・ローディング
			API①~④のリクエスト状態がtrueの場合、ローディングアイコンを表示する。


対応概要で表現しきれない、かつ、テストで漏れそうな機能の詳細（テストで必須な確認項目など）を記載する
また、必要に応じて以下を追記する（主に先方の確認を目的として）
	・既存の画面に変更が発生する場合、イメージがわかるような図の添付（画面キャプチャに変更点を追記するなど）
	・定義されていない機能について実装が必要な場合

各画面からカスタムできる要素
	画面①, ④
		株主優待名(例：SBIリーシングサービス株主優待)
		画面タイトル（例：2026年3月期末 株主優待）
		口座開設の注意コンポーネントの表示/非表示
		移管ガイドの表示/非表示
	画面③, ⑤
		株主優待名(例：SBIリーシングサービス株主優待)
		画面タイトル（例：2026年3月期末 株主優待申込確認）

エラーハンドリング
	クライアントバリデーション
		各入力フィールドの下に個別にエラーメッセージを表示
		申込コード、申込パスワード
			半角数字指定
				半角数字以外が入力：「半角数字で入力してください」
				指定外の桁数：「*文字で入力してください」
			半角英数字指定
				半角英数字以外が入力：「半角英数字で入力してください」
				指定外の桁数：「*文字で入力してください」
			※ *は指定の文字数
		リンク内ドキュメント
			リンク未押下の状態で「確認画面へ」ボタン押下：「株主優待のお申込みにはリンク内文書の確認が必要です。」
		「同意する」チェックボックス
			チェックボックス非押下の状態で「確認画面へ」ボタン押下：「お申込みには同意が必要です。」
	サーバーエラー
		口座開設未完了エラー(「お客様は口座開設未完了です、手続きを進めてください。」)の場合、口座開設画面(/signup)へリダイレクト
		以下エラーの場合はアラート表示し、通常ログイン画面(/signin)へリダイレクト
			顧客種別確認: 「本キャンペーンは個人口座向けです。」
			株主優待申込済み確認: 「お客様はすでにお申込み済みです。1口座あたり1回までとなっております。」
		以下エラーの場合はボタン上にインラインでエラーメッセージを表示
			該当株主優待の申込期間確認: 「お申込み受付期間外です。」
			該当する株主優待申込情報データの存在確認: 「※{0}または{1}が不一致です。再度入力をお願いします。」
			該当する株主優待申込情報データの申込可能確認: 「入力された{0}、{1}はすでにお申込み済です。」
