

# BPJ STG環境 DB接続手順書

### 前提条件

* macOSを利用していること
* `brew install cloudflared` が完了していること
* `stg-trd-key.pem` を入手済みであること

---

## 1. 鍵ファイルの配置と権限設定

初めて接続する場合や、鍵をダウンロードした直後に行う設定です。

1. **鍵ファイルの移動**: ターミナルで `stg-trd-key.pem` を `~/.ssh` に移動します。
```bash
mv ~/Downloads/stg-trd-key.pem ~/.ssh/

```


2. **権限の変更**: 鍵の権限を適切なもの（自分のみ読み書き可能）に制限します。
```bash
chmod 600 ~/.ssh/stg-trd-key.pem

```



---

## 2. SSH設定（config）の追記

`~/.ssh/config` に接続情報を記述します。

1. 設定ファイルを開きます。
```bash
nano ~/.ssh/config

```


2. 以下の内容を末尾に追記し、保存（`Ctrl+O` → `Enter`）して終了（`Ctrl+X`）します。
```text
Host stg-bpj-trddb-ro
  HostName stgtrdssh04.bitpoint.ne.jp
  User ec2-user
  IdentityFile ~/.ssh/stg-trd-key.pem
  GatewayPorts no
  LocalForward 43306 database-stgtrd-cluster.cluster-ro-cjupgrnbgg19.ap-northeast-1.rds.amazonaws.com:3306
  ProxyCommand /opt/homebrew/bin/cloudflared access ssh --hostname %h

```



---

## 3. SSHトンネル接続（ポートフォワード）

DBへアクセスする際は、必ずこの手順でトンネルを確立してください。

1. **コマンド実行**:
```bash
ssh stg-bpj-trddb-ro

```


* 初回のみ `Are you sure you want to continue connecting (yes/no/[fingerprint])?` と聞かれるので `yes` と入力します。


2. **ブラウザ認証**:
* 自動的にブラウザが立ち上がります。
* メールアドレスを選択し、OTP（ワンタイムパスワード）を入力して認証を完了させます。


3. **接続確認**:
* ターミナルに Amazon Linux のログインロゴが表示され、プロンプト（`[ec2-user@...]`）が出れば成功です。
* **注意**: このターミナルは**DB作業中は閉じないでください**。



---

## 4. DBeaverでの接続設定

ターミナルで接続が維持されている状態で、DBeaverを開きます。

* **接続ホスト**: `localhost`
* **ポート**: `43306`
* **ユーザー名**: `vct-takada.shotaro`
* **パスワード**: `aP2*R#YkqwW&M68ycnvf`
* **SSL**: 不要（チェックを外す）

---

### 運用上のポイント

* **接続終了時**: 作業が終わったらターミナルで `exit` と入力するか、ウィンドウを閉じてください。
* **接続できない場合**:
1. ターミナルで `ssh stg-bpj-trddb-ro` が正しく実行されているか確認。
2. ブラウザ認証を再度やり直す。
3. `lsof -i :43306` コマンドでポートが開放されているか確認。


---

### 次回以降の起動手順

#### 1. ターミナルでSSHトンネルを確立する

DBを利用する前に、まずターミナルを起動して以下のコマンドを入力します。

```bash
ssh stg-bpj-trddb-ro

```

* **認証プロセス**:
1. ブラウザが自動的に立ち上がり、ログイン画面が表示されます。
2. メールアドレスを入力し、届いたOTPコードを画面に入力して認証を完了してください。


* **確認**:
ターミナル上に `[ec2-user@...]` というプロンプトが表示されたら、ポートフォワード（トンネル）が正常に機能している証拠です。**このターミナルは作業中、絶対に閉じないでください。**

#### 2. DBeaverから接続する

ターミナルで接続が維持されている状態で、DBeaverを開き、既存の接続設定から「接続」を行ってください。

* 設定済みの設定（localhost:43306）を使用すれば、すぐにデータベースの中身を参照できます。

---

### 💡 運用上のヒント

* **ターミナルの最小化**:
邪魔な場合はターミナルを「閉じる」のではなく「最小化」しておくと、バックグラウンドで接続を維持したまま作業に集中できます。
* **接続が切れてしまった場合**:
もししばらく操作せずタイムアウトした場合や、PCをスリープから復帰させた後に接続エラーが出る場合は、ターミナルで一度 `exit` して、再度 `ssh stg-bpj-trddb-ro` を実行してください。

この2ステップで毎回スムーズに接続できるはずです。また何かあればいつでも聞いてくださいね！
