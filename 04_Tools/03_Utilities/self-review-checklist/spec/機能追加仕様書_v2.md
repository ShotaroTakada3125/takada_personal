# 機能追加仕様書 v2

---

## 概要

最小構成（v1）で実装した「セルフレビューチェックリストアプリ」に以下の機能を追加する。

### 追加機能一覧

| # | 機能名 | 概要 |
|---|--------|------|
| 1 | レビュー観点の追加・編集・削除 | チェック項目を動的に管理 |
| 2 | カテゴリーの追加・編集・削除 | カテゴリーを動的に管理 |
| 3 | タグの追加・編集・削除 | タグをマスター管理 |
| 4 | UI改善 | ヘッダー・サイドバー・モダンなデザイン |

---

## 1. レビュー観点の追加・編集・削除機能

### 1.1 機能要件

| ID | 機能名 | 説明 |
|----|--------|------|
| F-10 | 観点追加 | 選択中のカテゴリーに新しいチェック観点を追加 |
| F-11 | 観点編集 | 既存のチェック観点の内容・タグを編集 |
| F-12 | 観点削除 | チェック観点を削除（確認ダイアログあり） |
| F-13 | 観点並び替え | ドラッグ＆ドロップで表示順を変更（オプション） |

### 1.2 画面要素

```
【チェックリスト画面への追加要素】
- 「+ 観点を追加」ボタン（リスト下部）
- 各観点行に「編集」「削除」アイコンボタン（ホバー時表示）

【観点追加・編集モーダル】
- 観点内容（テキスト入力、必須、255文字以内）
- タグ選択（ドロップダウン、タグマスターから選択）
- 「保存」「キャンセル」ボタン
```

### 1.3 API仕様

| メソッド | エンドポイント | 説明 | リクエストボディ |
|----------|---------------|------|------------------|
| POST | `/api/items` | 観点を追加 | `{ category_id, content, tag_id }` |
| PUT | `/api/items/{id}` | 観点を編集 | `{ content, tag_id }` |
| DELETE | `/api/items/{id}` | 観点を削除 | - |

### 1.4 バリデーション

- content: 必須、1〜255文字
- tag_id: 任意（null許可）
- category_id: 必須、存在するカテゴリーID

### 2.1 機能要件

| ID | 機能名 | 説明 |

| F-21 | カテゴリー編集 | カテゴリー名を編集 |

### 2.2 画面要素

```

- カテゴリー一覧表示
- 「+ カテゴリーを追加」ボタン
- 各カテゴリーに「編集」「削除」アイコン（ホバー時表示）

【カテゴリー追加・編集モーダル】
- カテゴリー名（テキスト入力、必須、100文字以内）
- 「保存」「キャンセル」ボタン
```

### 2.3 API仕様

| メソッド | エンドポイント | 説明 | リクエストボディ |
|----------|---------------|------|------------------|
| POST | `/api/categories` | カテゴリーを追加 | `{ name }` |
| PUT | `/api/categories/{id}` | カテゴリーを編集 | `{ name }` |
| DELETE | `/api/categories/{id}` | カテゴリーを削除 | - |

### 2.4 バリデーション

- name: 必須、1〜100文字、重複不可

## 3. タグの追加・編集・削除機能

### 3.1 機能要件

| ID | 機能名 | 説明 |
|----|--------|------|
| F-30 | タグ追加 | 新しいタグを追加 |
| F-31 | タグ編集 | タグ名・色を編集 |
| F-32 | タグ削除 | タグを削除（使用中の観点からは外れる） |

### 3.2 画面要素

```
【タグ管理画面（設定モーダルまたはサイドパネル）】
- タグ一覧表示（名前＋色プレビュー＋使用数）
- 「+ タグを追加」ボタン
- 各タグに「編集」「削除」アイコン

【タグ追加・編集モーダル】
- タグ名（テキスト入力、必須、50文字以内）
- タグ色（カラーピッカーまたはプリセット選択）
- 「保存」「キャンセル」ボタン
```

### 3.3 データベース変更

```sql
-- タグマスターテーブル（新規追加）
CREATE TABLE tags (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE,
    color VARCHAR(7) DEFAULT '#2196f3'  -- HEXカラーコード
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- check_itemsテーブルの変更
-- tag(VARCHAR) → tag_id(INT, FK) に変更
ALTER TABLE check_items 
    DROP COLUMN tag,
    ADD COLUMN tag_id INT,
    ADD FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE SET NULL;
```

### 3.4 API仕様

| メソッド | エンドポイント | 説明 | リクエストボディ |
|----------|---------------|------|------------------|
| GET | `/api/tags` | タグ一覧取得 | - |
| POST | `/api/tags` | タグを追加 | `{ name, color }` |
| PUT | `/api/tags/{id}` | タグを編集 | `{ name, color }` |
| DELETE | `/api/tags/{id}` | タグを削除 | - |

### 3.5 バリデーション

- name: 必須、1〜50文字、重複不可
- color: 任意、HEXカラーコード形式（#RRGGBB）

### 3.6 初期データ

### 6.2 テーブル定義（更新版）

```sql
-- タグマスター（新規）
CREATE TABLE tags (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE,
    color VARCHAR(7) DEFAULT '#2196f3',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- カテゴリー（タイムスタンプ追加）
CREATE TABLE categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- チェック観点（tag → tag_id、sort_order追加）
CREATE TABLE check_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_id INT NOT NULL,
    content VARCHAR(255) NOT NULL,
    tag_id INT,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- チェック状態（タイムスタンプ追加）
CREATE TABLE check_states (
    id INT AUTO_INCREMENT PRIMARY KEY,
    item_id INT NOT NULL UNIQUE,
    is_checked BOOLEAN DEFAULT FALSE,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (item_id) REFERENCES check_items(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
