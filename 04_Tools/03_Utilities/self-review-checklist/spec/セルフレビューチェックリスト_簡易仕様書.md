## テーマ名

```
新人向けセルフレビュー観点チェックリストアプリ
```

---

## 1. 概要

### 1.1 何をするものか（30文字以内）

```
業務アクション前にチェック観点を確認・記録するWebツール
```

### 1.2 想定ユーザー

```
- ユーザー：新人メンバー
- 使用シーン：コードレビュー依頼、Slack送信などのアクション実行前のセルフチェック時
```

---

## 2. 画面構成

### 2.1 画面一覧

| # | 画面名 | 概要 |
|---|---|---|
| 1 | メイン画面 | カテゴリ選択＋チェックリスト表示＋進捗表示 |

> ※1画面で完結する設計。

### 2.2 画面要素

```
- ヘッダー：タイトル「セルフレビューチェックリスト」
- カテゴリ選択：ドロップダウン（コードレビュー / Slack送信 / メール送信）
- 進捗表示：「2/8 完了」形式のテキスト＋プログレスバー
- チェックリスト：
  - 各観点ごとに1行
  - チェックボックス ＋ 観点テキスト ＋ タグバッジ（例：[セキュリティ]）
- 完了メッセージ：全チェック時に「✅ 完了！」を表示
- リセットボタン：「リセット」（赤系、右下配置）
```

---

## 3. データの流れ

```
【初期表示】
1. 画面を開く
2. APIからカテゴリ一覧を取得
3. デフォルトで最初のカテゴリを選択
4. 選択カテゴリのチェック観点一覧＋チェック状態を取得
5. チェックリストを表示

【チェック操作】
1. ユーザーがチェックボックスをクリック
2. APIにチェック状態を送信（自動保存）
3. 進捗表示を更新
4. 全チェック完了なら「完了！」を表示

【カテゴリ切替】
1. ドロップダウンで別カテゴリを選択
2. 選択カテゴリのチェック観点一覧＋チェック状態を取得
3. チェックリストを再描画

【リセット】
1. リセットボタンをクリック
2. 確認ダイアログ表示
3. OKならAPIで現カテゴリのチェック状態を全クリア
4. チェックリストを再描画
```

## 4. 機能仕様

### 4.1 主要機能

| ID | 機能名 | 入力 | 処理 | 出力 |
|---|---|---|---|---|
| F-01 | カテゴリ一覧取得 | なし | APIからカテゴリ取得 | ドロップダウン項目 |
| F-02 | チェック観点一覧取得 | category_id | APIからチェック観点＋状態取得 | チェックリスト表示 |
| F-03 | チェック状態更新 | item_id, is_checked | APIでDB更新 | 進捗更新 |
| F-04 | 一括リセット | category_id | APIで該当カテゴリの状態クリア | チェックリスト再描画 |

### 4.2 API仕様（最小限）

| メソッド | エンドポイント | 説明 |
|---|---|---|
| GET | `/api/categories` | カテゴリ一覧取得 |
| GET | `/api/categories/{id}/items` | カテゴリに紐づく観点＋チェック状態取得 |
| PUT | `/api/items/{id}/check` | チェック状態を更新 |
| POST | `/api/categories/{id}/reset` | カテゴリのチェック状態を一括リセット |

### 4.3 エラー処理（最低限）

- API通信エラー → 「通信エラーが発生しました」表示
- それ以外 → コンソールログのみ（今回は詳細エラー処理しない）

## 6. データベース設計

### 6.1 テーブル定義

```sql
-- カテゴリ
CREATE TABLE categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL
);

-- チェック観点
CREATE TABLE check_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_id INT NOT NULL,
    content VARCHAR(255) NOT NULL,
    tag VARCHAR(50),
    FOREIGN KEY (category_id) REFERENCES categories(id)
);

-- チェック状態
CREATE TABLE check_states (
    id INT AUTO_INCREMENT PRIMARY KEY,
    item_id INT NOT NULL UNIQUE,
    is_checked BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (item_id) REFERENCES check_items(id)
);
```
