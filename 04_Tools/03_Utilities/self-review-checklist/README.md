# セルフレビューチェックリストアプリ v2

新人向けのセルフレビュー観点チェックリストWebアプリケーションです。

## 概要

業務アクション（コードレビュー依頼、Slack送信、メール送信など）の実行前に、セルフチェックすべき観点を確認・記録するためのツールです。

## 新機能（v2）

- ✨ **カテゴリー管理**: カテゴリーの追加・編集・削除
- ✨ **チェック観点管理**: 観点の追加・編集・削除
- ✨ **タグ管理**: タグの追加・編集・削除（色設定可能）
- ✨ **UI改善**: ヘッダー・サイドバーを追加したモダンなデザイン
- 📱 レスポンシブ対応（基本）

## 技術スタック

- **フロントエンド**: HTML + バニラJavaScript + CSS
- **バックエンド**: Python FastAPI
- **データベース**: MySQL 8.0
- **インフラ**: Docker Compose

## 起動方法

### 1. Docker Composeでバックエンドとデータベースを起動

```bash
cd self-review-checklist
docker-compose up -d
```

### 2. フロントエンド用のHTTPサーバーを起動

```bash
cd frontend
python3 -m http.server 3001
```

または

```bash
cd frontend
/usr/bin/python3 -m http.server 3001
```

### 3. ブラウザでアクセス

```
http://localhost:3001
```

## 使い方

### 1. カテゴリーの管理

- **追加**: サイドバーの「+」ボタンからカテゴリーを追加
- **編集**: カテゴリー名の横の鉛筆アイコンをクリック
- **削除**: カテゴリー名の横のゴミ箱アイコンをクリック（確認ダイアログが表示されます）

### 2. チェック観点の管理
### カテゴリー
| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| GET | `/api/categories` | カテゴリー一覧取得 |
| POST | `/api/categories` | カテゴリー追加 |
| PUT | `/api/categories/{id}` | カテゴリー編集 |
| DELETE | `/api/categories/{id}` | カテゴリー削除 |
| GET | `/api/categories/{id}/items` | 観点一覧＋チェック状態取得 |
| POST | `/api/categories/{id}/reset` | チェック状態リセット |

### チェック観点
| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| POST | `/api/items` | 観点追加 |
| PUT | `/api/items/{id}` | 観点編集 |
| DELETE | `/api/items/{id}` | 観点削除 |
| PUT | `/api/items/{id}/check` | チェック状態更新 |

### タグ
| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| GET | `/api/tags` | タグ一覧取得 |
| POST | `/api/tags` | タグ追加 |
| PUT | `/api/tags/{id}` | タグ編集 |
| DELETE | `/api/tags/{id}` | タグ削除
### 3. タグの管理

- **タグ設定を開く**: ヘッダー右上のタグアイコンをクリック
- **追加**: タグ管理画面で「タグを追加」ボタンをクリック
- **編集**: 各タグの鉛筆アイコンをクリック
- **削除**: 各タグのゴミ箱アイコンをクリック
- **色設定**: タグ追加・編集時にカラーパレットから選択

### 4. 進捗確認とリセット

- 画面上部に進捗（例: 2/8 完了）とプログレスバーが表示されます
- 全チェック完了時に「完了！」メッセージが表示されます
- 「リセット」ボタンで現在のカテゴリーのチェックをクリア

## API仕様

| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| GET | `/api/categories` | カテゴリ一覧取得 |
| GET | `/api/categories/{id}/items` | カテゴリに紐づく観点＋チェック状態取得 |
| PUT | `/api/items/{id}/check` | チェック状態を更新 |
| POST | `/api/categories/{id}/reset` | カテゴリのチェック状態を一括リセット |

## 停止方法

### フロントエンドサーバーの停止

ターミナルで `Ctrl + C` を押してHTTPサーバーを停止します。

### Docker Composeの停止

```bash
cd self-review-checklist
docker-compose down
```

データを保持したまま停止する場合（推奨）:

```bash
docker-compose down
```

データも削除する場合:

```bash
docker-compose down -v
```

## チェック観点の追加・編集

初期データは `backend/init.sql` に記載されています。  
カテゴリやチェック観点を追加・編集する場合は、このファイルを編集してからDocker Composeを再起動してください。

```bash
docker-compose down -v  # データを削除
docker-compose up -d    # 再起動
```
